  2. 负数记得输出'-'

1002. A+B for Polynomials

像归并排序一样把两个数字序列合并就好了，若x次数不同，把次数小的先加入到结果中；次数相同系数相加，若相加后系数不为0则加入到结果中。

1003. Emergency

  1. 最短路径题
  2. 先求起点为源点的单源最短路径dis
  3. 再求终点为源点的单源最短路径rdis
  4. 对于一条端点为u和v，长为c的边若dis[u]+rdis[v]+c==dis[dst]||dis[v]+rdis[u]+c==dis[dst]那么这条边就是最短路径上的边，做上标记。
  5. 对标记好的最短路径上的边DFS，维护医疗队最多人数和总路径数即可。
  6. 嘛，其实也可以不用两次单源最短路径，在求dis的时候维护一个当前节点在最短路径中前一个节点列表即可。

1004. Counting Leaves

从根节点DFS，带上当前的level，维护每层叶子节点个数即可。

1005. Spell It Right

  1. 输入的数字很大，用char num[101];吧。
  2. 计算每一位上数字的和
  3. 写个数字到英文单词的映射，把和的每一位输出即可。

1006. Sign In and Sign Out

用string读入，字符串比较时间，维护最大和最小。

1007. Maximum Subsequence Sum

  1. 最大子段和
  2. 如果输入全部为负数，输出0，并输出第一个和最后一个数字。
  3. 维护当前子段的起始位置，当sum<0时更新起始位置，sum>ans时更新答案。

1008. Elevator

求和。

1009. Product of Polynomials

  1. 多项式乘法，记录每项次数的系数。
  2. 输出系数不为0的项的系数。

1010. Radix

  1. 注意溢出，建议使用long long。
  2. 直接枚举进制radix是会超时的。
  3. 试试二分radix吧，上界是目标的十进制表示。
  4. 按下面这种姿势会WA第三个,把mid=(low+high)/2;移到下面才过，好费解。 
    while(low<=high)
{
	mid=(low+high)/2;
	int t=check(n2,mid,target);
	if(t>=0)
	{
		high=mid-1;
		if(t==0)
			if(res1==-1||res1>mid)
				res1=mid;
	}
	else
		low=mid+1;
	//mid=(low+high)/2;
}

1012. The Best Rank

  1. 排序题
  2. 分别按平均分和三门课成绩排序，计算每种排序中的排名，记录最靠前的。
  3. 注意分数相同的情况，比如100 99 99 80，对应排名为1 2 2 4，而不是 1 2 2 3.

1013. Battle Over Cities

  1. 求无向图的连通分量数
  2. 直接DFS，对未标记的节点DFS，把所有能访问到的节点都标记上。
  3. 执行了几次DFS就有几个连通分量，答案为连通分量数-1.

1021. Deepest Root

  1. 先用DFS求一次图的连通分量数，大于1则直接按题目要求输出。
  2. 因为题目所求根节点的度必为1，所以维护所有节点的度数。
  3. 对度数为1的节点DFS，记录遍历树最长深度以及对应的节点。

1023. Have Fun with Numbers

  1. 数字比较大，不要试图使用int或者long long，用高精度乘法吧。
  2. 统计原来的数中每个数字出现次数，统计乘2后每个数字出现次数，对比得到结果。

1024. Palindromic Number

  2. 本身就是回文数的直接输出。
  3. 不停用高精度加法，直到得到回文数为止。

1032. Sharing

两种方法

  1. 分别遍历两个链表，若最终字符相同，则说明有共同部分，令dif=abs(lenA-lenB)，令两指针再次指向各自的头，较长的先移动dif步，然后同时向前移动，直到两指针指向的地址相同。
  2. 维护两个栈，分别遍历两个链表，一边遍历一边把节点压入栈中，遍历结束后比较栈顶，若栈顶相同，同时弹栈，直到栈顶的节点不同为止。

1033. To Fill or Not to Fill

  1. 很容易误认为是DP的贪心题。
  2. 可以想象成用不同的油漆涂抹一条道理，按油漆价格排序，先处理便宜的。
  3. 用油漆从油漆桶（加油站）的位置开始涂抹，跳过已经被涂抹的路段，直到油漆用完。
  4. 检查整条道路是否都被涂上了颜色。
  5. 计算每种油漆涂抹的距离，计算费用。

1037. Magic Coupon

  1. 把优惠券从小到大排序，把商品按从小到大排序。贪心。
  2. 两个指针分别指向优惠券和商品的头，一边相乘把乘积加到结果，一边向前移动。直到某个指针指向的数大于0.
  3. 两个指针分别指向优惠券和商品的末尾，一边相乘把乘积加到结果，一边向前移动。直到某个指针指向的数小于0.
  4. 输出结果。

1038. Recover the Smallest Number

  1. 把所有的数字按照字典序从大到小排序。
  2. 把所有数字按排序的顺序输出，注意前导0 。
  3. 注意答案为0的情况。

1039. Course List for Student

  1. 用map映射人名到ID。
  2. vector维护每个课程注册的学生ID。
  3. 按课程ID从小到大处理，把每个课程的ID添加到注册这个课程的学生的vector中。
  4. 读入查询，输出结果。
  5. 把student数组开到40010老是越界，乘10以后就过了。

1043. Is It a Binary Search Tree

  1. 按照二叉搜索树前序遍历得到的序列插入可以重建原树？待证明。
  2. 总之按照输入序列按两种方法插入一颗空树，然后前序遍历得到的树，看是否与输入相同。
  3. 若能构造原树，输出后序遍历序列。

1044. Shopping in Mars

  1. 从左往右扫描，维护一个区间所有数的和，区间右端点不断右移。
  2. 若当前的和大于所求的数，区间左端点向右移动，直到区间的和小于等于所求的数。
  3. 扫描的时候记录离所求的数最接近的区间和，并记录区间左右端点。

1045. Favorite Color Stripe

  1. 把喜欢的数字序列中的每个数字map到它的下标（或者说等级）。
  2. 把输入序列的每个数字利用上面的map转成数字的等级，不在喜欢的数字中的数字忽略掉。
  3. 问题转化成最长上升子序列问题。
  4. L的范围比较大，最好用O(NlogN)的解法，即维护一个栈，若当前数字比栈顶元素大，压入栈中；否则二分当前的栈，找到第一个大于当前数字的位置，用当前数字替换掉，最后输出栈的大小。
